## 一.显示方式
### `display` 属性
指定元素的显示方式。

|值|说明|宽高属性|宽高默认值|
|--------|----|----|----|
|`none`|元素不显示|无效|0|
|`inline`|元素显示成内联元素|无效|内部占用宽高，其中文字按行算宽高。|
|`inline-block`|元素显示成内联块元素|有效|内部占用宽高，其中文字按整块算宽高。|
|`block`|元素显示成块状元素|有效|100%宽度，内部占用高。即使指定了宽度，它也会占用一整行。|
|`inline-flex`|元素本身为内联块元素，内部按照 `flex` 布局排列。|有效|内部占用宽高，其中文字按整块算高。|
|`flex`|元素本身为块元素，内部按照 `flex` 布局排列。|有效|100%宽度，内部占用高。即使指定了宽度，它也会占用一整行。|

## 二.定位
### 标准流
默认情况下，元素都是按照标准流进行排布。

特点：
+	从左到右、从上到下按顺序摆放。
+	默认情况下，元素之间不存在层叠现象。

### `position` 属性
指定元素定位的方式

|值|说明|定位参考物|百分比参考物|滚动方式|
|--------|----|----|----|----|
|`static`(默认)|静态定位|按照标准流定位|父元素|内容随滚动条滚动而滚动|
|`relative`|相对定位|父元素|父元素|内容随滚动条滚动而滚动|
|`absolute`|绝对定位|`static`定位以外的第一个父元素。（如果没有则为浏览器视图区窗口）|`static`定位以外的第一个父元素。（如果没有则为浏览器视图区窗口）|内容随滚动条滚动而滚动|
|`fixed`|固定定位|浏览器窗口|浏览器窗口|内容不随滚动条滚动而滚动|

说明：
1.	**定位参考物** 指使用 `left` `right` `top` `bottom` 进行定位时相对的元素。

2.	**百分比参考物** 指 `width` `height`  `left` `right` `top` `bottom` 属性使用百分比时，计算实际数值参照的元素。$实际值 = 百分比 \times 参考物的宽或高$

3.	`absolute`和`fixed`会使内容脱离标准流。


### `left` `right` `top` `bottom` 属性
对于非静态定位元素，设置相对参考物偏移的属性。

!> 这些属性会使 `width` 和 `height` 属性失效。可通过 `margin:auto` 来使它重新生效。

### `z-index` 属性
对于非静态定位元素，指定其显示的优先级。

## 三.浮动
### `float` 属性
|值|说明|
|----|----|
|`left`|元素左浮动|
|`right`|元素右浮动|
|`none`|元素不浮动|

### `float` 规则
1.	元素一旦浮动后就脱离标准流，朝着向左或向右的方向移动，直到自己的边界紧贴包含块（一般是父元素）或者其他浮动元素的边界为止。
2.	浮动元素不会与行内级元素和含有文本的内容重叠，其内容会被浮动元素推出。如`inline-block`、`inline`或者块级元素的文本内容。
3.	浮动元素不参与非BFC父元素宽高的计算。这容易导致 **高度塌陷** 问题。

### `clear` 属性
用于清除浮动。

|值|说明|
|----|----|
|`left`|清除左浮动|
|`right`|清除右浮动|
|`none`|清除左右浮动|

将 具有`clear`属性的元素放置与浮动元素前后，即可清理浮动。（可以在父元素上添加伪元素）

###  几种清除浮动的方法
1.	固定父容器高度。 不够灵活。
2.	将 具有`clear`属性的元素放置与浮动元素前后。（通常的做法是使用伪元素）
3.	在浮动元素后面添加 `<br clear="all">`。
4.	触发BFC机制。例如给父级元素加个 `overflow:hidden` 属性。

## 四.内容溢出

### `overflow` 属性
用于安排内容溢出。

!>同时要限制元素的宽/高

|值|说明|
|----|----|
|`visitible`|显示内容溢出的部分|
|`scroll`|值能阻止内容溢出，但会增加出滚动条，通过拉动滚动条可以浏览所有内容|
|`hidden`|隐藏掉内容溢出的部分，溢出的内容将不可见|
|`auto`|如果内容溢出被限制，则会添加一个滚动条，使超出的内容可以通过滚动展示出来|

### 容器属性--对齐

#### `justify-content`属性

决定了 容器元素 在主轴上的对齐方式。

（不考虑 `margin`属性）

| 值                   | 对齐方式（→为主轴正方向） | 容器元素之间的距离 | 容器元素与边界的距离         |
| -------------------- | ------------------------- | ------------------ | ---------------------------- |
| `flex-start`（默认） | 左对齐                    | 0                  | 任意值                       |
| `flex-end`           | 右对齐                    | 0                  | 任意值                       |
| `center`（常用）     | 居中对齐                  | 0                  | 任意值                       |
| `space-between`      | 两端对齐                  | 相等               | 0                            |
| `space-around`       | 两端对齐                  | 相等               | 等于容器元素之间的距离的一半 |
| `space-evenly`       | 两端对齐                  | 相等               | 等于容器元素之间的距离       |

#### `align-items`和`align-content`

这两个属性决定了 容器元素在交叉轴上的对齐方式。

其中 `align-items` 针对单行元素，`align-content` 针对多行元素。

（不考虑 `margin`属性）

| 值                   | 对齐方式（↓为交叉轴正方向） |
| -------------------- | --------------------------- |
| `flex-start`（默认） | 顶部对齐                    |
| `flex-end`           | 底部对齐                    |
| `center`（常用）     | 居中对齐                    |
| `stretch`            | 拉伸                        |

### 元素属性

#### `order`属性

用于指定元素的显示顺序。数字越大，顺序越靠后。

**默认值** 0

#### `flex-grow`属性

元素放大的比例，0表示有剩余空间时不放大。

如果两个元素的增长比例为 2:1，那么这两个元素占用空间之比为 2:1。

**默认值** 0

#### `flex-shrink`属性

定义项目缩小的比例，0表示有没有剩余空间时不缩小。

如果所有项目的`flex-shrink`属性都为1，当空间不足时，都将等比例缩小。

**默认值** 1

#### `flex-basis`属性

定义了在分配多余空间之前，项目占据的主轴空间。

**默认值** auto（项目本来的大小）

#### `flex` 属性

是 `flex-grow`、`flex-shrink`、`flex-basis`的缩写（按顺序）。

```css
flex:0 1 auto;
```

#### `align-self`属性

允许单个项目有与其他项目不一样的对齐方式，可覆盖`align-items`属性。其取值等同于`align-items`。

**默认值** auto(继承容器属性)

## 六.BFC

BFC 即 Block Formatting Contexts (块级格式化上下文)。 **具有 BFC 特性的元素可以看作是隔离了的独立容器，容器里面的元素不会在布局上影响到外面的元素，并且 BFC 具有普通容器所没有的一些特性。**

### 触发BFC的条件

满足下面条件其一即可触发BFC：
+	 `body` 元素
+	浮动元素: `float` 除 `none` 以外的值
+	绝对或固定定位
+	显示方式为`inline-block`、`table-cells`、`flex`的元素
+	`overflow` 除了 `visible` 以外的值 (`hidden`、`auto`、`scroll`)

### BFC的特色
1.	同一个 BFC 下的元素，外边距会发生折叠。
2.	BFC 元素会计算内部浮动元素的宽高。 **这个特色可以清除浮动。**
3.	阻止元素被浮动元素覆盖。
